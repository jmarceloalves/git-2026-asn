1. Inicializar um repositório git
	git init .

2. Listar arquivos:
	ls
	ls -a #somente arquivos ocultos

3. Listar em qual diretório estou:
	pwd

4. Criar arquivos no bash
	touch arquivo.txt

5. A pasta .git é a responsável por fazer todo o controle de versão local

6. Buscar o status do git
	git status

7. Adicionar um arquivo no git
	git add <nome_arquivo.extensao>

8. Fazer o commit no git
	git commit -m "mensagem de commit"

9. Comandos para configurar o responsável pelo commit
  git config --global user.email "you@example.com"
  git config --global user.name "Your Name"

10. Enxergar todos os commits realizados
	git log

11. Adicionando todos os arquivos no controle do git
	git add . #Extremamente perigoso

12. Comando para remover arquivos adicionados mas não comitados
	git reset #Limpa toda a branch

13. O que é uma branch? Uma branch é uma bifurcação que isola o estado de um projeto em um determinado
	momento no tempo. 
	Comando para criar uma branch:
	git checkout -b <nome_da_branch>

14. Para trocar de branch execute o comando:
	git checkout <nome_da_branch>

15. Comando para saber quantas branchs existem:
	git branch
